Interactive Gamelan — Idea

Goal

Create a **playable gamelan** on the web with **near‑zero strike latency**, suitable for teaching, practice, and demos.

Sound Design

We approximate metallophone/gong timbre using a simple modal synthesis model with five inharmonic partials. To ensure immediate response, each note is pre‑rendered into an AudioBuffer at load.

Performance

  • AudioContext uses latencyHint: "interactive".
  • Pre‑rendered buffers → AudioBufferSourceNode.start() on strike; no real‑time graph building overhead.
  • Limiter prevents clipping during fast rolls or chords.
